# 👥 لیست اعضای تیم – وظایف و دانش فنی

این سند یک نمای کلی از همه اعضای تیم، نقش‌ها، مسئولیت‌ها و سطح دانش فعلی آن‌هاست.  
هر عضو موظف است پروفایل شخصی خود را در `dev/Exercises/your-name/profile.md` تکمیل کند.

> 🎯 ارشدها تجربه بیشتری دارند و مسئولیت راهنمایی و هماهنگی را بر عهده دارند – نه برتری مقامی.  
> 🌱 اعضای جونیور در حال یادگیری هستند و انتظار می‌رود با پرسش و تمرین، سریعاً رشد کنند.

## 🧩 شرح وظایف بر اساس نقش

### مهدی قورچی (لیدر فنی)
- معماری کلی پروژه (ساختار بک‌اند، ارتباط با AI، زیرساخت)
- تصمیم‌گیری نهایی در مورد تکنولوژی‌ها و ابزارها
- بررسی Pull Requestهای مرتبط با بک‌اند، AI و دوآپس
- راه‌اندازی و نگهداری از سرور و زیرساخت (همراه رضا)
- داوری در تخلفات فنی و عدم توافق

### رضا قادری (ارشد فرانت و دوآپس)
- طراحی معماری فرانت (React/Next.js)
- مدیریت جلسات هفتگی (زمان‌بندی، دستور جلسه، پیگیری)
- پیگیری گزارش‌های هفتگی و وضعیت تسک‌ها
- همکاری در استقرار و دوآپس (به همراه مهدی)

### ارین امیری (مدیر مستندات و بک‌اند/AI)
- ثبت تمام تسک‌ها و تکالیف در چک‌لیست
- جمع‌آوری گزارش‌های هفتگی و یادآوری به افراد
- ثبت اخطارها و تخلفات (طبق قوانین تیم)
- مستندسازی فرآیندها، تصمیمات و نقشه‌راه‌ها
- توسعه ماژول‌های AI و بک‌اند (همکاری با مهدی)

### اعضای جونیور (امیر، سینا، مهدی خانزاده، مهدی فتحی)
- انجام تسک‌های تعیین شده توسط ارشدها
- ارسال گزارش روزانه و هفتگی
- پرسیدن سوال و رفع اشکال با کمک دیگران
- تلاش برای یادگیری طبق نقشه‌راه حوزه خود
- مشارکت در پروژه‌های تمرینی گروهی

## 🛠 دانش فنی مورد انتظار در پایان فاز وب (۵ ماه)

### بک‌اند ( مهدی، ارین، امیر، سینا)
- Python پیشرفته (OOP, decorators, generators)
- Django (مدل، ویو، فرم، احراز هویت، ORM)
- Django REST Framework (API)
- FastAPI (حداقل برای سرویس AI)
- PostgreSQL (طراحی، کوئری، ایندکس)
- Redis (کش، صف)
- Git و GitHub گردش کار پایه
- Docker (ساخت image، docker-compose)

### فرانت‌اند (رضا، مهدی خانزاده، )
- HTML5/CSS3 (Flex, Grid، ریسپانسیو)
- JavaScript (ES6+، async/await، DOM)
- React (hooks, context, routing)
- Next.js (SSR/SSG، routing)
- TypeScript (مقدماتی)
- TailwindCSS
- تست فرانت (Jest, React Testing Library)

### هوش مصنوعی (مهدی، ارین)

- HuggingFace Transformers (استفاده از مدل‌های آماده)
- PyTorch (مقدماتی: تنسور، مدل ساده، حلقه آموزش)
- FastAPI (برای سرویس کردن مدل)

### دوآپس (مهدی، رضا)
- لینوکس سرور (Ubuntu)
- Docker و Docker Compose
- Nginx (reverse proxy)
- GitHub Actions (CI/CD ساده)
- مانیتورینگ پایه (htop, logs)

## 📌 نکات مهم

- هر عضو موظف است فایل `profile.md` خود را در `dev/Exercises/your-name/` تکمیل و به‌روز نگه دارد.
- ارشدها حق تغییر تسک‌ها و اولویت‌ها را با مشورت یکدیگر دارند.
- اعضای جونیور می‌توانند بدون ترس سوال بپرسند – تنها راه یادگیری سوال کردن است.
- در صورت عدم توانایی در انجام تسک، قبل از ددلاین به ارین یا مسئول مربوطه اطلاع دهید.
