# 🔀 راهنمای سریع گیت (Git) برای تیم

این صفحه فقط دستورهایی رو میگه که عملاً بهشون نیاز داریم. اگر کامل بلدی، می‌تونی به عنوان مرجع سریع استفاده کنی. اگر تازه شروع کردی، قدم به قدم برو جلو.

> 🤝 **یک اصل:** گیت دوست ماست. قرار نیست ما رو اذیت کنه. اگر به مشکلی خوردی، نترس – همه ما از همین اشتباهات یاد گرفتیم.

---

## 🧰 دستورهای روزانه (برای شروع کار)

| کاری که می‌خواهم انجام دهم | دستور |
|----------------------------|--------|
| دریافت آخرین تغییرات دیگران | `git pull` |
| دیدن وضعیت فعلی (چه فایل‌هایی تغییر کرده) | `git status` |
| اضافه کردن یک فایل مشخص به مرحله commit | `git add filename` |
| اضافه کردن همه فایل‌های تغییر یافته | `git add .` |
| ثبت تغییرات (commit) با یک پیام کوتاه | `git commit -m "پیام مفید"` |
| فرستادن تغییرات به گیت‌هاب | `git push` |

---

## 🌿 کار با شاخه (Branch)

| کاری که می‌خواهم انجام دهم | دستور |
|----------------------------|--------|
| دیدن شاخه فعلی | `git branch` |
| ساختن یک شاخه جدید و رفتن به آن | `git checkout -b feature/xyz` |
| رفتن به یک شاخه دیگر (مثلاً develop) | `git checkout develop` |
| ادغام یک شاخه در شاخه فعلی | `git merge feature/xyz` |

> **یادآوری:** قبل از ساختن شاخه جدید، حتماً `git pull` بزن تا شاخه `develop` یا `main` به‌روز باشد.

---

## 📝 استاندارد پیام commit

- **به فارسی یا انگلیسی** (در کل تیم مهم نیست، فقط یکدست باشه).  
- **شروع با فعل:** `add`، `fix`، `update`، `remove`، `refactor`.  
- **کوتاه و توصیفی** (حداکثر ۵۰ کاراکتر).  
- مثال‌های خوب:  
  - `add weekly report template`  
  - `fix bug in user login`  
  - `update roadmap-back.md`  
- مثال‌های بد:  
  - `تغییرات`  
  - `asdf`  
  - `درست شد`

## 🩹 حل تداخل (Conflict)

### چه زمانی پیش می‌آید؟
وقتی دو نفر同一个 فایل را تغییر دهند و هر دو بخواهند تغییراتشان را push کنند، گیت نمی‌داند کدام را قبول کند.

### قدم‌ها:
1. `git pull` را بزن. گیت بهت می‌گوید «CONFLICT» در بعضی فایل‌ها.  
2. فایل مورد نظر را باز کن. خط‌هایی شبیه این پیدا می‌کنی:
   ```
   <<<<<<< HEAD
   کد خودت
   =======
   کد فرد دیگر
   >>>>>>> branch-name
   ```
3. تصمیم بگیر کدام نسخه را نگه داری (یا هر دو را ترکیب کن). قسمت‌های اضافی (`<<<<`, `====`, `>>>>`) را پاک کن.  
4. فایل را ذخیره کن.  
5. `git add .`  
6. `git commit -m "resolve conflict in filename"`  
7. `git push`

> اگر واقعاً گیج شدی: `git merge --abort` همه چیز را به قبل از conflict برمی‌گرداند. بعد از یکی از اعضای تیم کمک بگیر.

## 🗑️ اشتباهات رایج و راه حل

| اشتباه | راه حل |
|--------|--------|
| `git push` دادم، اما می‌گوید کار نمی‌کند (rejected) | اول `git pull` بزن، بعد دوباره `git push` کن. |
| تغییرات را کامیت کردم، اما یادم رفت یک فایل اضافه کنم | `git add filename` بعد `git commit --amend --no-edit` (مراقب باش – فقط برای آخرین کامیت). |
| پیام کامیت را اشتباه نوشتم | `git commit --amend -m "پیام جدید"` |
| خواستم شاخه را عوض کنم ولی می‌گوید تغییرات دارم (تغییرات کامیت نشده) | یا `git add . && git commit -m "..."` یا اگر نمی‌خواهی نگه داری: `git stash` (بعداً با `git stash pop` برمی‌گرده). |

## 📌 خلاصه گام‌های روزانه (برای ۹۰٪ مواقع)

```bash
git pull                         # ۱. دریافت آخرین تغییرات
git checkout -b feature/my-work  # ۲. ساختن شاخه جدید (اختیاری)
# ... کار کردن و ویرایش فایل‌ها ...
git add .                        # ۳. اضافه کردن همه تغییرات
git commit -m "توضیح کار انجام شده"
git push origin HEAD             # ۴. فرستادن به گیت‌هاب
```

سپس در گیت‌هاب، یک **Pull Request** بساز و یکی از ارشدها را reviewer بگذار.
