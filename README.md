## 📌 قوانین کلیدی (خلاصه)

- ✅ **گزارش روزانه:** حتی ۵ خط در فایل شخصی (`.md` یا `.txt`) بنویس امروز چه آموختی.  
- ✅ **گزارش هفتگی:** تا جمعه شب کاملش کن و در گروه بفرست (قالب آماده است).  
- ✅ **جلسه هفتگی:** جمعه‌ها ساعت ۱۷، حداکثر ۱ ساعت – حضور الزامی (غیبت موجه = به ارین بگو).  
- ✅ **تسک‌های الزامی:** توسط لیدر یا ارشدها تعیین می‌شوند و برای همه اجباری هستند.  
- ❌ **توهین، حاشیه‌سازی، بی‌تعهدی:** هشدار → اخطار → رأی تیم.

> متن کامل قوانین در [`team-Structure.md`](team-Structure.md) نوشته شده.

## 🚀 شروع سریع برای اعضا

1. **کلون کردن ریپو**  
   ```bash
   git clone https://github.com/your-org/team-repo.git
   ```
2. **پوشه شخصی بساز**  
   ```bash
   cd dev/Exercises/
   mkdir your-name
   ```
3. **پروفایل خود را تکمیل کن**  
   از [`member/profile-template.md`](member/profile-template.md) یک کپی بردار، اطلاعاتت را بنویس و در پوشه‌ی خودت ذخیره کن.
4. **نقشه راه حوزه‌ات را بخوان** (`learning/`)
5. **هر روز تمرین کن و گزارش بنویس**  
6. **جمعه گزارش هفتگی را پر کن و در گروه بفرست**

## 🧑‍🤝‍🧑 چگونه همکاری کنیم؟

- **گروه اصلی (تلگرام/اسلک):** برای پیام‌های روزانه و رفع اشکال سریع.  
- **جلسه هفتگی:** جمع‌بندی پیشرفت‌ها و تعیین اولویت هفته بعد.  
- **مستندات:** هر تغییری در ریپو باید با Pull Request و تأیید یکی از ارشدها انجام شود.  
- **پروژه اصلی:** کدها در `main/main-pr/` قرار می‌گیرند و روی شاخه `develop` توسعه می‌یابند.

> راهنمای کامل گیت و کار با ریپو در فایل‌های [`work-with-repository.md`](work-with-repository.md) و [`git-basics.md`](git-basics.md) است.
